命令注入

常用命令注入函数

  • 在PHP中，可以实现执行外部程序或函数的命令执行函数

    eval()，assert()， system()，shell_exec()，exec()，passthru()，“(与shell_exec功能相同)

  • 函数原型
    • string system(string command, int &return_var)

      ：command 要执行的命令

      ：return_var 存放执行命令的执行后的状态值
    • string exec (string command, array &output, int &return_var)

      ：command 要执行的命令

      ：output 获得执行命令输出的每一行字符串

      ：return_var 存放执行命令后的状态值
    • void passthru (string command, int &return_var)

      ：command 要执行的命令

      ：return_var 存放执行命令后的状态值
    • string shell_exec (string command)

      ：command 要执行的命令

DVWA命令注入之shell_exec()

  • 环境：windows系统、DVWA平台

low级别

源代码

  • 源代码函数stristr()和php_uname()的分析

    函数stristr(string,search,before_search)和php_uname（mode）函数，

    • 第一个函数的作用:是搜索字符串在另一字符串中的第一次出现。before_search为可选参数默认值为 “false” 的布尔值。如果设置为 “true”，它将返回 search 参数第一次出现之前的字符串部分。该函数的返回值为返回字符串的剩余部分（从匹配点），如果未找到所搜索的字符串，则返回 FALSE。
    • 第二个函数的作用:是返回运行php的操作系统的相关描述，参数mode可取值

      ”a” （此为默认，包含序列”s n r v m”里的所有模式）

      ”s ”（返回操作系统名称）

      ”n”（返回主机名）

      ”r”（返回版本名称）

      ”v”（返回版本信息）

      ”m”（返回机器类型）

      因此这段代码的含义为判断该操作系统为哪种操作系统并按操作系统的不同执行其对应的Ping命令

  • 分析源代码
    知对输入的参数没有做任何防御处理，可直接通过命令执行函数shell_exec()，执行相关系统命令,从而存在漏洞，给攻击者可乘之机，获取系统相关系统信息

  • 执行流程：

    在表单输入IP地址，点击Submit按钮；通过$_POST[ ‘Submit’ ]获得Submit请求，再通过$_REQUEST[ ‘ip’ ]获取输入的IP地址；再利用shell_exec()函数执行命令，执行的结果回显到当前网页中。通过命令的拼接从而可以利用漏洞，进行注入攻击

命令拼接

&&：第一个命令执行成功才会执行第二个命令
  • 尝试拼接命令(127.0.0.1&&dir)
  • 随意输入然后用&&进行拼接命令，都不能执行
&：不管第一个命令是否执行成功都会执行第二个命令
  • 尝试拼接命令(127.0.0.1&dir)

    观察知，第一命令ping和第二个dir都执行
  • 尝试拼接命令(aaa&dir)

    观察知，第二个命令dir成功执行
|（管道符）：只执行第二个命令
  • 输入（www.baidu.com | whoami）

    可以看到，只执行了第二个名令
cmd1 || cmd2：cmd1执行失败才执行cmd2
  • 仅执行了第一个命令
  • 第一个失败执行第二个命令
;（分号）拼接命令

经过测试，在linux系统下，能够成功执行多个命令

在windows下，不能成功执行
